## 0. 한 장 요약 (TL;DR)

- **시작(5월)**: GPT 이미지 생성으로 캐릭터 스틸을 만들어보고 "**오, 이거 되겠다**"에서 불붙었다.
  근데 **영상(애니메이션)은 완전히 다른 과정**이었다 — 스틸은 GPT로 됐지만 움직이는 건 별개의 어려운
  파이프라인(Sora→Veo i2v). 사진 → GPT images.edit(캐릭터) → Veo i2v로 **손으로 한 편씩**, 사람이 taste를
  잡고 CLI가 실행.
- **문제**: (a) 스토리 품질 병목이 **상류 작문**에 있었고, (b) AI 티 나는 영상은 초기 채널에서 무시당했고,
  (c) 리뷰어가 결함을 눈감아주고, (d) 맥 인프라가 불안정했다.
- **대응**: 단일 LLM 호출을 **작가→감독→촬영→편집→검수→PD** 스튜디오 조직으로 **분화**하고,
  4편/일 A/B 실험 + 밴딧 측정으로 **탐색을 시스템화**하고, 헛것(캐릭터 환각)을 막는 **3층 지식 시스템**을
  세우고, LLM을 못 믿는 검수는 **결정론 게이트**로 바꿨다.
- **핵심 태도**: 실패를 지우지 않고 **남긴다**. "롤백한 것"이 다음 사람의 지도가 된다.
- **현재(7월)**: 렌더 품질은 안정권 → 남은 병목은 다시 **연출/캡션 작문** → 콘티(cue-sheet) 에이전트로 분화 중.
  인프라 근본해결(GCP 이전)은 준비 완료·미실행.

## 3. 에이전트의 분화 (Agent Differentiation)

> **핵심 서사**: 하나의 LLM 호출이 실제 영상 스튜디오의 분업 조직으로 갈라졌다.
> "각자가 오케스트라가 잘 안 된다"(PD)는 진단이 이 분화의 방향타였다 — 개별 에이전트 튜닝이 아니라
> **오케스트레이션**이 병목이었다.

### 3.1 분화의 계보

```

                         │  병목: i2v 엔진 개선이 나쁜 작문을 못 고침
                         ▼
[5/30]  ┌── Writer (Opus, 3-pass: 초안→자기비판→수정)  … 스토리 전용
        └── Director (1-pass) … 촬영(shot/camera/light) + 프롬프트 조립
                         │
[6/13]  + Cameraman = Editor 로 승격  … 의도 vs 실제 footage 판단, 편집 셰이핑
        + 상류 피드백 루프 (recaption / different_clip 재제안)
        + Macro Reviewer  … 교차-에피소드 신선도(공개된 영상 기준)

        + PD-taste Selector  … PD의 과거 선택/veto에서 학습하는 랭커
                         │

        + 지식 에이전트: pet_profile(VLM 관찰) · knowledge(모르면 PD에 질문)
                         │
[6/7]   + Launch Orchestrator  … 레인×슬롯 편성·자동예약
```

### 3.3 "조용한 에이전트" — 결정론 게이트

LLM을 못 믿는 판단은 코드가 대신한다(에이전트의 또 다른 형태):
`_temporal_grounding_gate`(시점) · `_preachy_caption_gate`(도사체) · `_rf_temporal_coherence`(era/존재)
· `_false_first_water_gate`(첫수영 거짓) · `_concept_lexical_collision`(재탕) · nape/무꼬리/얼굴 게이트.
**원칙: 프롬프트 텍스트만으론 리뷰어를 못 고친다 — 신호를 계산해 먹여라.**

### 3.4 협업 루프
**board(Slack 봇, 24/7 자율) ↔ CLI(대화형)** 는 `notes/progress_log.md` 공용 로그로 소통한다.
세션 시작 시 최근 로그를 읽고 상대가 한 일 위에서 이어간다 — 두 에이전트가 따로 놀지 않게 하는 접합부.

### 3.5 메타 층 — "규칙 퇴적"을 알아차리고 스킬로 (6/14~6/23)
분화된 건 에이전트만이 아니다. 어느 순간 PD가 알아챘다: 프롬프트 `.md`들이 **rule-base로 퇴적**되고
있었다 — "PD <날짜>: '…'" 날짜박힌 밴드에이드가 쌓이고 **서로 모순**되고 일반화가 안 돼, PD가 같은
지적을 반복해야 했다. 대응 = **프로세스를 스킬로 코드화**(콘텐츠 에이전트가 아니라, 시스템이 자기 규칙을
어떻게 바꾸는지를 다스리는 규율 층):
- **prompt-authoring** (6/14, `bf09421`) — 규칙은 **원칙→왜→어떻게→예시**로 쓰고, 날짜블록을 덧붙이지
  말고 **통합·supersede**(provenance는 git이 보관). 프롬프트 `.md`와 auto-메모리 둘 다 적용(전사 아닌 synthesize).
- **pipeline-change-impact** (6/14, `bf09421`) — 한 변경을 **양 레인 전체**(Writer→Director→Cameraman→
  Giri, AV+RF)에 전파·검증·보고. "Writer 고치고 Director/RF 잊음" 방지.
- **review-learning** (6/15, `68b37c6`) — PD 수동 리뷰 피드백 → 일관성 감사 후 **durable 규칙으로 일반화**.
- **giri-update** (6/23, `a4e64b7`) — 생성기만 고치고 **검수기(Giri)를 안 맞추던** 드리프트 → 검수기를 lockstep으로.

**의미**: 파이프라인은 에이전트 수만 는 게 아니라, **프롬프트·규칙이 어떻게 진화하는지를 다스리는 규율**이
생겼다. 이 회고를 쓰는 방식(메모리 synthesize, 핸드오프)도 그 산물이다.

## 6. 관통하는 교훈 (Meta-Lessons)

1. **$50 렌더는 $5 이터레이션이 아니다.** $0.04 스틸/$0 캡션-salvage로 먼저 검증, PD OK 후 유료 렌더.
2. **생성기만 고치면 샌다.** 최종 캡션 단계 + 검수기(Giri)가 **lockstep**으로 같이 움직여야.
3. **프롬프트 텍스트로는 리뷰어를 못 고친다.** 신호를 **코드로 계산**해 boolean을 먹여라(결정론 게이트).
4. **Additive 게이트는 순증 악화할 수 있다.** 새 변경은 실제 도움 검증 후에만 유지.
5. **인프라 탓 전에 내 최근 변경부터.** 파일 mtime을 렌더 시각과 대조.
6. **모든 DB 테이블엔 독자가 있어야.** 죽은 파이프(background_refs, trends)가 PD 지시를 조용히 버린다.
7. **실패 서명을 구분하라.** 즉시 ENOSPC=디스크 / 600s 실행=경합 / 무한블록=락.
8. **AV는 매 프레임을 생성한다.** 실제 펫 채널은 촬영→드리프트 0. 일부 드리프트는 구조적 — 실사감/안정성
   트레이드오프를 받아들여라.
9. **증상수정 ≠ 재발방지.** 에피소드를 고치는 것과 게이트를 고치는 것을 명확히 구분해 보고.
10. **자동화의 진짜 병목은 오케스트레이션.** 개별 에이전트 튜닝이 아니라 **분업과 접합**.
11. **스틸 ≠ 영상.** GPT 이미지로 캐릭터가 "되겠다" 싶어도, 움직이는 건 완전히 다른(더 어려운) 파이프라인.
12. **규칙은 통합, 밴드에이드는 퇴적.** 날짜박힌 규칙을 덧붙이면 모순이 쌓인다 → 원칙-우선 + supersede,
    그리고 그 규율 자체를 **스킬로** 박아 시스템이 자기 규칙 진화를 다스리게.
13. **사람+에이전트 git 협업은 아직 미해결 마찰.** 브랜치→main FF로 완화했으나 세션충돌·스크래치 오염이
    남음 — production truth(main)를 깨끗이 지키는 규율이 계속 필요(궁극엔 단일 배포경로).
14. **레인은 동시에 살아있어야 한다.** 한 레인 파는 동안 방치된 레인은 조용히 회귀한다("RF 박살") —
    자동 배치는 둘 다 요구한다. "고친 것"이 아니라 "둘 다 최소선 위"가 성공 기준.
15. **랴니 마킹이 프로젝트 최대 난제였다.** AV는 매 프레임을 생성하니 헛마킹이 불가피 → **엄격 게이트(오탐)와
    느슨함(헛것) 사이**를 ref 비교 + 어드바이저리 캡 + 컷별 self-heal로 겨우 균형. 실제 펫 채널엔 없는 문제.
16. **배경/원근은 두 달 내내 어려웠다.** 3D 모델링 → 정밀스틸 → 첫+끝 interp 모두 2D/평면으로 무너졌다.
    실사감의 답은 **정교한 합성이 아니라 실제 소스**(실사진 ref + i2v) — 배경 안정성은 아직 완전 정복 못 함.
